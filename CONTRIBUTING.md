# Contributing to GLYPH

We welcome contributions to GLYPH! Here are some guidelines:

## Issues
- Check existing issues before creating new ones
- Provide detailed descriptions with screenshots if applicable
- Include device info and Android version

## Pull Requests
- Fork the repository
- Create a feature branch (`git checkout -b feature/amazing-feature`)
- Commit your changes (`git commit -m 'Add amazing feature'`)
- Push to the branch (`git push origin feature/amazing-feature`)
- Open a Pull Request

## Code Style
- Follow Kotlin conventions
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing
- Test on multiple device sizes
- Verify animations on different refresh rates
- Check theme switching functionality
- Test widget placement and editing

## Feature Requests
Please open an issue with:
- Clear description of the feature
- Use cases and benefits
- Any design mockups or references
